Java Fundamentals

This chapter introduces the core building blocks of Java programming, including syntax, data types, control structures, methods, and basic input/output. Perfect for beginners or those revising the basics.

1.1 Java Syntax & Structure

Java is a strongly-typed, object-oriented language with a clear structure. Every program requires a class, and the main method serves as the entry point. The file name must match the public class name.


public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Welcome to Java Mastery!");
    }
}

Key Point: The main() method is required for program execution and must be declared as public static void.

1.2 Variables & Data Types

Java supports primitive and reference data types for storing different kinds of values. Primitive types include integers, floating-point numbers, booleans, and characters, while reference types include objects like Strings.

  • Primitive Types: byte, short, int, long, float, double, boolean, char
  • Reference Types: String, arrays, custom classes

int age = 25;
double salary = 50000.50;
boolean isActive = true;
char grade = 'A';
String name = "Alice";

Warning: Always initialize variables before use to avoid NullPointerException for reference types or undefined behavior for primitives.

1.3 Control Structures

Control structures manage the flow of execution using conditionals (if-else, switch) and loops (for, while, do-while).

Control Flow Diagram
if-else → decision tree; loops → iterative execution

int score = 85;
if (score >= 90) {
    System.out.println("Grade: A");
} else if (score >= 80) {
    System.out.println("Grade: B");
} else {
    System.out.println("Grade: C");
}

for (int i = 1; i <= 5; i++) {
    System.out.println("Count: " + i);
}

1.4 Methods

Methods are reusable blocks of code that perform specific tasks. They improve code organization and modularity.


public class Calculator {
    public static int add(int a, int b) {
        return a + b;
    }

    public static void main(String[] args) {
        int result = add(5, 3);
        System.out.println("Sum: " + result); // Output: Sum: 8
    }
}

Key Point: Use appropriate access modifiers (public, private) to control method visibility.

1.5 Basic Input/Output

Java provides classes like Scanner for input and System.out for output.


import java.util.Scanner;

public class InputDemo {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        System.out.print("Enter your name: ");
        String name = scanner.nextLine();
        System.out.println("Hello, " + name + "!");
        scanner.close();
    }
}

Object-Oriented Programming

Learn the core principles of OOP: classes, objects, encapsulation, inheritance, polymorphism, and abstraction. These concepts are fundamental to writing robust Java code.

2.1 Classes & Objects

Classes define blueprints for objects, which are instances containing state (fields) and behavior (methods).


public class Student {
    String name;
    int age;

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public void displayInfo() {
        System.out.println("Name: " + name + ", Age: " + age);
    }
}

Student student = new Student("Alice", 20);
student.displayInfo(); // Output: Name: Alice, Age: 20

2.2 Encapsulation

Encapsulation protects data by using private fields and public methods to control access.


public class BankAccount {
    private double balance;

    public BankAccount(double balance) {
        this.balance = balance;
    }

    public void deposit(double amount) {
        if (amount > 0) balance += amount;
    }

    public double getBalance() {
        return balance;
    }
}

2.3 Inheritance

Inheritance allows a subclass to inherit properties and methods from a superclass.


public class Vehicle {
    protected String brand;

    public Vehicle(String brand) {
        this.brand = brand;
    }

    public void honk() {
        System.out.println("Honk!");
    }
}

public class Car extends Vehicle {
    private String model;

    public Car(String brand, String model) {
        super(brand);
        this.model = model;
    }
}

2.4 Polymorphism

Polymorphism enables objects to be treated as instances of their parent class, with method overriding for runtime behavior.


public class Animal {
    public void makeSound() {
        System.out.println("Generic sound");
    }
}

public class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Woof!");
    }
}

2.5 Abstraction

Abstraction hides implementation details using abstract classes and interfaces.


public abstract class Shape {
    public abstract double area();
}

public class Circle extends Shape {
    private double radius;

    public Circle(double radius) {
        this.radius = radius;
    }

    @Override
    public double area() {
        return Math.PI * radius * radius;
    }
}

Exception Handling

Learn to manage errors and exceptional conditions using try-catch blocks, custom exceptions, and best practices.

3.1 Exception Types

Exceptions are divided into checked (compile-time) and unchecked (runtime) categories.

Exception Hierarchy
Throwable → Error, Exception → IOException, RuntimeException

try {
    int[] arr = new int[5];
    System.out.println(arr[10]);
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("Error: " + e.getMessage());
}

3.2 Try-Catch-Finally

Use try-catch to handle exceptions and finally for cleanup.


try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Cannot divide by zero");
} finally {
    System.out.println("Cleanup code");
}

3.3 Throws Keyword

Declare exceptions that a method might throw using throws.


import java.io.*;

public class FileReader {
    public String readFile(String file) throws IOException {
        BufferedReader reader = new BufferedReader(new FileReader(file));
        String line = reader.readLine();
        reader.close();
        return line;
    }
}

3.4 Custom Exceptions

Create custom exceptions by extending Exception.


public class InvalidAgeException extends Exception {
    public InvalidAgeException(String message) {
        super(message);
    }
}

public class Person {
    public void setAge(int age) throws InvalidAgeException {
        if (age < 0) throw new InvalidAgeException("Age cannot be negative");
    }
}

File I/O

Learn to read from and write to files using Java's I/O classes.

5.1 Reading Files

Use BufferedReader for efficient file reading.


import java.io.*;

try (BufferedReader reader = new BufferedReader(new FileReader("input.txt"))) {
    String line;
    while ((line = reader.readLine()) != null) {
        System.out.println(line);

    e.printStackTrace();
}

try (BufferedWriter writer = new BufferedWriter(new FileWriter("output.txt"))) {

Threads have states like NEW, RUNNABLE, BLOCKED, and TERMINATED.

Thread Lifecycle
NEW → RUNNABLE → BLOCKED/WAITING → TERMINATED

Use ExecutorService for thread pool management.

JDBC & Databases

Connect Java applications to databases using JDBC for data management.

9.1 JDBC Basics

JDBC connects Java to relational databases.


import java.sql.*;

Connection conn = DriverManager.getConnection("jdbc:mysql://localhost:3306/db", "user", "pass");
Statement stmt = conn.createStatement();
ResultSet rs = stmt.executeQuery("SELECT * FROM users");

9.2 Prepared Statements

Use PreparedStatement for secure queries.


import java.sql.*;

PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM users WHERE id = ?");
pstmt.setInt(1, 1);
ResultSet rs = pstmt.executeQuery();

9.3 Transaction Management

Manage database transactions for data integrity.


import java.sql.*;

conn.setAutoCommit(false);
try {
    PreparedStatement pstmt = conn.prepareStatement("UPDATE accounts SET balance = ?");
    pstmt.setDouble(1, 1000.0);
    pstmt.executeUpdate();
    conn.commit();
} catch (SQLException e) {
    conn.rollback();
}

Design Patterns

Learn reusable solutions to common software design problems.

10.1 Singleton Pattern

Ensures a class has only one instance.


public class Singleton {
    private static Singleton instance;

    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

10.2 Factory Pattern

Creates objects without specifying the exact class.


public interface Animal {
    void speak();
}

public class Dog implements Animal {
    public void speak() { System.out.println("Woof!"); }
}

public class AnimalFactory {
    public Animal createAnimal(String type) {
        if ("Dog".equals(type)) return new Dog();
        return null;
    }
}

10.3 Observer Pattern

Defines a one-to-many dependency between objects.


import java.util.*;

public class Subject {
    private List observers = new ArrayList<>();

    public void addObserver(Observer o) {
        observers.add(o);
    }

    public void notifyObservers() {
        for (Observer o : observers) {
            o.update();
        }
    }
}

public interface Observer {
    void update();
}

10.4 Strategy Pattern

Defines a family of algorithms and makes them interchangeable.


public interface PaymentStrategy {
    void pay(int amount);
}

public class CreditCardPayment implements PaymentStrategy {
    public void pay(int amount) {
        System.out.println("Paid " + amount + " via Credit Card");
    }
}

10.5 Decorator Pattern

Adds responsibilities to objects dynamically.


public interface Coffee {
    double cost();
}

public class SimpleCoffee implements Coffee {
    public double cost() { return 2.0; }
}

public class MilkDecorator implements Coffee {
    private Coffee coffee;

    public MilkDecorator(Coffee coffee) {
        this.coffee = coffee;
    }

    public double cost() {
        return coffee.cost() + 0.5;
    }
}
